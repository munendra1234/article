Let us see why in the first place we needed arrays.

Suppose you wanted to store your marks in a test. How would you do it?

I think you’ll say that you assign your marks to a variable like : int marks = 88. Yes, that’s correct.

Now, what if you had to store marks for your 3 friends?

It’s a similar condition, so we can have 3 variables.

int marksA = 75; int marksB = 91; int marksC = 82;

I know so many big words. Let's break it down.

Approach :

Collection of items :

This means that under a single name, you’ll be able to store marks of your whole class, thus, you need not make 40 different variables.

Same type :

Items of the same type means that you can only store values of the same data type (integer, character, strings, decimal numbers, boolean values, etc.)

You can’t store strings in the array of integers and vice-versa.

We’ll talk about continuous memory allocations later in this article.

Now that we are little familiar with arrays, let’s see what an array looks like.

This is an array named marks, which contains marks of 5 students.

Arrays in Java are 0-indexed. It means that the first item in the array is at index 0. The second item is at index 1 and so on.

Exercise :

If an array has n items, what will be the index of the last item ?

You can make a diagram if you want and don’t peek at the answer till you have one yourself.

The last item would be the nth item, thus, it’s index will be n-1. But the size of the array is still n, as it contains n items.

We are doing pretty well, it seems like arrays are easy to learn.

Now is the right time to learn about “How to make an array ?”.

int [] marks = new int[5];

This line makes an array somewhere in the memory that can contain 5 integer elements and names the array as marks. Since the default value of int is 0, every array element is assigned 0 automatically.

If we break down this line :

int [] marks; this is the declaration of array

marks = new int[5]; When we use ‘new’ to create an array, Java reserves space in memory for it. This process is called memory allocation.

Now that we have made an array, how do we add value to it?

Using square brackets, this is how.

marks[0] = 100; >

Through this line, we assign 100 to the first item (at index 0) of array marks.

Let’s put it all together, what we’ve learned till now.

It might not be possible for you to assign values like this to array elements. So, we use loops for this purpose.

We run the loop from index 0 to index n-1 (n is length of array).

import java.util.*; public class Main{ public static void main(String[] args) { Scanner scn = new Scanner(System.in); int[] marks = new int[5]; for (int i = 0; i < marks.length; i++) { marks[i] = scn.nextInt(); } } }
java; true";

marks.length gives us the length of the array, in this case 5.

Using this kind of for loop, we can directly assign the input from the user to the array.

Now the last thing that’s left is, how to access these array elements. It is very easy.

If you want to access the third array element i.e. 96 from this array, we’ll simply write : int a = marks[ 2 ]. Now, the variable a contains 96.

import java.util.*; public class Main{ public static void main(String[] args) { Scanner scn = new Scanner(System.in); int[] marks = new int[5]; for (int i = 0; i < marks.length; i++) { marks[i] = scn.nextInt(); } for (int i = 0; i < marks.length; i++) { System.out.println(marks[i]); } } }
java; true";

This code will take 5 integer inputs for array marks and print them on the console.

You’ll agree with me that when you create a variable, it is stored somewhere in the memory. Somewhere in the universe!! Same is the case with arrays.

Contiguous memory locations :

When you write new int[7], memory equivalent to 7 integer variables is assigned to the array. Contiguous memory locations means that these memory locations are in continuation, not scattered throughout the memory.

Applications on Array :
  • Arrays store data elements of the same data type.
  • They are used to implement other data structures like Stacks, Queues, Heaps, etc.
Disadvantages of using arrays:
  • You can’t change the size of an array, once declared.
