Search in a 2D sorted array

Welcome back reader!!

So, how’s your experience with 2D arrays so far?

I know sometimes they can be a lot to take in but you’ll have to agree with me that the concepts are interesting and even easy once you get them.

So, moving forward, in this article we’ll be searching for a given element in a 2D array. But, it's no ordinary array, it’s sorted in the rows as well as the columns.

Important Links : Problem Link, Question Video , Solution Video

Let’s start with the simplest approach possible.

LINEAR SEARCH :

So, this is a task for you. You have to visit every element in the matrix sequentially. If you find the required element, you have to print it’s row & column and if the element is not present in the matrix, you have to print “Not Found”.

>

I know you can do this, it’s really easy.

Can you guess the time-complexity for this approach ? Just take 1 minute and think it through.

Since, you are traversing each and every element of the matrix, the time-complexity will be O(m*n), as in a matrix of n rows and m columns, you have m*n elements.

Let’s talk about the space-complexity, after we’re done writing the code, as it will provide a clearer picture.

So, let’s see the solution for this one.

import java.util.*; public class Main { public static void main(String[] args) throws Exception { Scanner scn = new Scanner(System.in); int n = scn.nextInt(); int m = scn.nextInt(); int[][] arr = new int[n][m]; for (int row = 0; row < n; row++) { for (int col = 0; col < m; col++) { arr[row][col] = scn.nextInt(); } } int data = scn.nextInt(); scn.close(); for (int row = 0; row < n; row++) { for (int col = 0; col < m; col++) { if(arr[row][col] == data){ System.out.println( row ); System.out.println( col ); return; } } } System.out.println("Not Found"); } }
java; true";

So, here we traverse through every element, and as soon as the required element is found, we print it’s coordinates and return.

Since, we don't use any extra data structures here, the space-complexity is constant, O( 1 ).

Let us now move to a more finer approach, through which we can actually use the special feature of the 2D-array.

Do you remember the Binary Search approach, which we applied for 1D arrays ?

If you don’t, you can take a quick recap from this video.

We are going to use a somewhat similar approach. So, let's start.

Step 1 :

We can enter the matrix from any of the 4 corners, let’s see which ones are preferable. Remember, that we only can move along the row or column, no diagonals are entertained.

For this given matrix, we have to find Data = 12. Through this diagram, we’ll see why the top-left and bottom-right corners are not preferable.

The base of binary search was that you have reached a particular element and since the array is sorted, the elements on the left would be smaller and the right ones would be larger. According to what you have to search, either you go rightwards and discard / ignore the left array or vice-versa.

We have to do the same here.

Case 1 :

We enter from the top left corner i.e. 2. Since, 2 < 12, we need to move to the side where elements are greater than 2, so that we can reach 12. But in both adjacent cells, we have elements greater than 2 i.e. 3 and 6. Now, we are indecisive as to where to proceed, thus we are stuck.

Case 2 :

We enter from the bottom right corner i.e. 50. Since, 50 > 12, we need to move to the side where elements are smaller than 50, so that we can reach 12. But in both adjacent cells, we have elements smaller than 50 i.e. 40 and 25.

Now, we are indecisive as to where to proceed, thus we are stuck.

This is why the top-left and bottom-right corners are not preferable to enter the matrix in this question.

Now, let's quickly analyse the case of top-right and bottom-left cells. If we are standing at the top right cell which is 8, we’ll want to go to a side where elements are greater than 8, so that we can reach 12. On the left side of 8, all elements are smaller than 8 indicating we can’t find 12 there. Whereas, the elements below 8 are larger. So we can move there.

Last case, If we are standing at the bottom left cell which is 23, we’ll want to go to a side where elements are smaller than 23, so that we can reach 12. So, we move upwards as the elements on the right are greater than 23.

Through this analysis, we found out that either we enter from top-right or bottom-left.

In this article, we discuss the approach where we enter from top right.

Let us understand this technique.

  1. We stand at 8 (top-right), to the left we have smaller elements and downwards we have larger ones. Since 15>8, we move downwards and land on 18.
  2. Standing at 18, to the left again we have smaller elements and downwards we have larger elements. 18>15, so we move towards smaller elements i.e. left and land on 12.
  3. To the left of 12, there are smaller elements, thus we move downwards, since 15>12.
  4. Now, standing at 20, to the left we have smaller elements, thus we move left since 15< 20.

And VOILA !! We have reached our destination.

For a better explanation, you can always refer to the solution video. Now, try a few dry runs for different elements yourself and then attempt to write the code.

import java.util.*; public class Main { public static void main(String[] args) throws Exception { Scanner scn = new Scanner(System.in); int n = scn.nextInt(); int[][] arr = new int[n][n]; for (int i = 0; i < n; i++) { for (int j = 0; j < n; j++) { arr[i][j] = scn.nextInt(); } } int data = scn.nextInt(); int row = 0; int col = n - 1; while (row < n && col >= 0) { if (arr[row][col] == data) { System.out.println(row); System.out.println(col); return; } if (arr[row][col] > data) { col--; } else { row++; } } System.out.print("Not Found"); } }
java; true";

This is our final code. Initially, row = 0 and col = n-1, representing the top right element. Since we only move down or left, the row might become >= total rows or col might become < 0. To prevent these situations, we put this condition in while.

If we find the element, we print it’s coordinates and return. If after traversing the whole while loop, we still do not find the element, then we print “Not Found”.

Time Complexity :

Whereas, the time-complexity drops from O(m*n) to O(m+n).

Space Complexity :

The space-complexity of this approach is constant, O(1).
